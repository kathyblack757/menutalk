# 菜根谭 MenuTalk — 需求文档

## 项目背景

在跨文化旅游中，饮食是感知异国文化最直接的窗口之一。然而，尽管获取饮食这一环节往往不存在门槛，但在"饮食语言"上往往存在巨大的认知鸿沟。例如，中国菜名中广泛存在视觉隐喻，如"蚂蚁上树"、"狮子头"，单纯的字面翻译常令不懂中文的游客困惑甚至产生误解。

现有的翻译工具（网易翻译、Papago等）多为工具属性，缺乏对食材构成、烹饪逻辑及文化背后的叙事性呈现，无法满足游客在点单时对文化结构理解的潜在需求。

## 目标用户

在苍蝇小馆就餐的用户：
- 餐厅本身很可能是手写菜单
- 未被主流点评软件收录
- 店员不懂外语
- 用户无法通过常规渠道了解菜品

## 核心功能

### 1. 多语言菜单翻译
- 拍摄菜单照片，AI自动识别所有菜品
- 支持多语言互译
- 不仅翻译菜名，更解释菜品构成

### 2. 菜品信息卡片
每道菜展示：
- 菜品参考照片
- 菜品名称（原文 + 翻译）
- 食材构成（图标 + 文字）
- 可能过敏原（⚠️标识）
- 菜品介绍（文化背景/做法简述）
- 口味信息：辣度、热量、甜度、咸度

### 3. 智能点单
- 左右滑动浏览菜品（左滑喜欢，右滑跳过）
- 购物车收集喜欢的菜品
- 生成当地语言点单话术
- AI语音朗读点单文案
- 总价计算 + 汇率换算

### 4. 历史记录
- 保存过往拍摄记录
- 按时间倒序展示
- 点击查看历史菜品详情

### 5. 个人设置
- 界面语言设置
- 目标翻译语言设置
- 本国货币设置
- 饮食禁忌/过敏原设置

## 用户流程

```
拍摄菜单 → AI识别所有菜品 → 卡片浏览（左右滑动）→ 购物车 → 生成点单文案 → AI朗读
```

1. 首页（默认拍摄Tab）：打开相机/上传菜单照片
2. 卡片浏览页：以用户拍摄照片为高透底图，逐张展示菜品卡片
   - 顶部进度："1/51"
   - 左滑 = 不喜欢 → 不加入购物车
   - 右滑 = 喜欢 → 加入购物车
3. 购物车：收集右滑菜品，显示数量和单价
4. 点单文案：生成当地语言点单话术，AI朗读，总价换算

## 非功能需求

- H5移动端独立网页，任何浏览器可打开
- 响应式设计，适配主流手机屏幕
- 主要颜色：金色/棕色系
- 风格：简洁、直观、轻量
- 数据存储：浏览器localStorage（MVP阶段）
